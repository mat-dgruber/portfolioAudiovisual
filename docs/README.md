# 📚 Documentação do Projeto

Este diretório contém toda a documentação estratégica, técnica e de planejamento do **Portfolio Audiovisual**. Estes documentos servem como a "única fonte de verdade" para o desenvolvimento e manutenção do projeto.

## 🗂️ Índice de Documentos

### 1. [PRD (Product Requirements Document)](./PRD.md)

Define a visão do produto, o problema que resolve, o público-alvo e os requisitos funcionais (MVP). É o ponto de partida para entender o **porquê** do projeto.

### 2. [Arquitetura do Sistema](./ARCHITECTURE.md)

Detalha a stack tecnológica, a estrutura de pastas, o fluxo de dados e o mapeamento de componentes. Explica o **como** o sistema foi construído tecnicamente.

### 3. [Plano de Implementação](./PLAN.md)

Um roadmap detalhado dividido em fases, desde o setup inicial até o deploy final e otimizações de SEO/Performance. Útil para acompanhar o progresso do desenvolvimento.

### 4. [Regras de IA (AI_RULES)](./AI_RULES.md)

Diretrizes específicas para assistentes de codificação (como Claude, Gemini, etc.) sobre padrões de código, estilo de escrita e restrições técnicas específicas deste repositório.

## 🎨 Design & Assets

- **Estado do Design:** Consulte o arquivo `design-state.md` na raiz do projeto para ver tokens de cores, tipografia e referências visuais.
- **Protótipos:** Referências a mockups e wireframes (se aplicável) estão descritas no PRD.

## 🤝 Contribuição e Manutenção

Ao adicionar novas funcionalidades:

1.  Atualize o **PRD** se a funcionalidade alterar o escopo do produto.
2.  Atualize a **Arquitetura** se novos serviços, modelos ou bibliotecas forem adicionados.
3.  Garanta que as **Regras de IA** permaneçam atualizadas para auxiliar futuros ciclos de desenvolvimento.
